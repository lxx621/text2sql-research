# 项目规范
1. 对话语言：所有交互和输出内容必须使用中文，确保清晰易懂的沟通。
2. 文件操作：实现高效的文件分段读写功能，支持大文件处理，确保操作过程中资源占用合理，读写效率优化。
3. 系统环境：所有操作和开发工作必须在Windows操作系统环境下进行，确保兼容性和可执行性。
4. 参考资源：仅允许读取"vanna"目录（该目录为GitHub克隆的vanna代码库），严格禁止对该目录内的任何文件或子目录进行修改、删除或添加操作。
5. 文档管理："docs"目录需按照文档类型（包括但不限于需求文档、设计文档、用户手册、API文档等）进行明确分类存放，建立清晰的目录结构，确保文档有序且易于查找和管理。
6. Python环境要求：如涉及Python开发，必须创建独立的虚拟环境，所有项目依赖包必须安装在该虚拟环境中，且在执行任何Python代码前必须确保已正确切换至该虚拟环境。
