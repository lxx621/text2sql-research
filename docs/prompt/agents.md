# 项目规范
1. 对话语言：所有交互和输出内容必须使用中文，确保清晰易懂的沟通。
2. 文件操作：实现高效的文件分段读写功能，支持大文件处理，确保操作过程中资源占用合理，读写效率优化。
3. 系统环境：所有操作和开发工作必须在Windows操作系统环境下进行，确保兼容性和可执行性。
4. 参考资源：仅允许读取"vanna"目录（该目录为GitHub克隆的vanna代码库），严格禁止对该目录内的任何文件或子目录进行修改、删除或添加操作。
5. 文档管理："docs"目录需按照文档类型（包括但不限于需求文档、设计文档、用户手册、API文档等）进行明确分类存放，建立清晰的目录结构，确保文档有序且易于查找和管理。
6. Python环境要求：如涉及Python开发，必须创建独立的虚拟环境，所有项目依赖包必须安装在该虚拟环境中，且在执行任何Python代码前必须确保已正确切换至该虚拟环境。
7. 时间获取规范：在所有系统组件和功能模块中，必须统一使用"Time mcp"接口或服务获取时间数据。确保时间戳的一致性、准确性和同步性，避免使用本地系统时间或其他非标准时间获取方式，以消除跨环境时间差异问题。
8. 技术参考标准：当开发过程中遇到技术方案不确定、架构设计存疑或需要确认最佳实践时，必须使用"Context7 mcp"工具获取指定版本或最新的官方技术文档、架构指南和编码规范。所有技术决策必须基于该工具提供的官方指导进行，确保与项目技术标准保持一致。
9. 文件编码要求：项目中所有创建和修改的文件（包括但不限于源代码文件、配置文件、资源文件、文档文件等）必须使用UTF-8字符编码格式保存，且不得包含BOM(Byte Order Mark)。在IDE和构建工具中应统一配置默认编码为UTF-8，确保跨平台、跨编辑器的文件兼容性。
10. 代码注释规范：为所有生成的代码（包括但不限于函数、类、接口、复杂逻辑块、配置项等）添加符合项目规范的详细注释。注释内容必须包含：功能用途说明、参数列表及各参数含义、返回值类型及意义、核心逻辑说明、异常处理说明（如适用）、重要算法或业务规则解释。注释应清晰易懂，便于后续维护和团队协作。
