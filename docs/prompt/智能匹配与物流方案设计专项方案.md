# 9. AI 智能匹配与物流方案设计专项方案

## 专项定位

- AI 智能匹配是本项目的核心能力，应服务于物流方案生成、费用测算、线路优化、承运商推荐和风险提示。
- 系统不以单一模型替代业务规则，而采用"规则引擎 + 运筹优化 + 机器学习 + 知识库 + 人工复核"的组合式智能。
- 所有关键推荐结果应具备推荐理由、数据依据、置信度和人工确认机制。

## 9.1 业务目标
AI 智能匹配与物流方案设计模块的目标，是在交易商提交运输需求后，自动生成成本最优、时效最优、综合推荐、公转铁推荐和总包运输建议等多套可比选方案，并为每套方案提供成本构成、线路路径、承运商建议、时效预估、风险提示和推荐理由。

## 9.2 适用场景

| 场景 | 业务价值 |
|------|----------|
| 首次运输询价 | 快速形成多套方案，减少人工反复询价。 |
| 重点线路比选 | 比较不同路线、运输方式和承运商组合的成本与时效。 |
| 公转铁评估 | 判断公路直达和公铁联运的成本、时效、政策适配度。 |
| 总包运输设计 | 为大吨位、多批次、多节点运输形成整体组织方案。 |
| 承运商择优 | 基于信用、履约、价格、线路覆盖和可用运力形成推荐排序。 |
| 异常风险预判 | 识别轨迹偏离、超时、节点拥堵、单证缺失和承运商履约风险。 |
| 运营与监管分析 | 形成运价趋势、运力热力、公转铁统计和异常运输研判。 |

## 9.3 输入数据模型

AI 模块的数据输入分为需求数据、基础数据、历史数据、实时数据和规则数据五类。为保障推荐质量，应在一期同步建设数据质量评分和缺失字段提醒机制。

| 数据类别 | 关键字段 |
|----------|----------|
| 运输需求数据 | 起点、终点、吨位、煤种、发运时间、到货时限、装卸要求、特殊约束。 |
| 基础资源数据 | 物流节点、线路、站点、货场、车辆、承运商、司机、服务区域。 |
| 历史业务数据 | 历史成交价、历史线路、合同履约、运输时效、异常记录、投诉记录。 |
| 实时过程数据 | 车辆定位、铁路状态、运力可用情况、承运商报价、节点拥堵、天气和交通管制。 |
| 规则与政策数据 | 计费规则、准入规则、黑白名单、线路限制、公转铁政策、平台运营规则。 |

## 9.4 输出方案结构

| 输出项 | 说明 |
|--------|------|
| 方案类型 | 成本最优、时效最优、综合推荐、公转铁推荐、总包运输建议。 |
| 线路路径 | 起点、途经节点、中转节点、终点及各段运输方式。 |
| 费用构成 | 公路、铁路、短倒、装卸、仓储、等待、风险和其他附加费用。 |
| 预计时效 | 装货、发运、中转、到达、卸货和总体预计时长。 |
| 承运商建议 | 推荐承运商、推荐原因、信用等级、可用运力和历史表现。 |
| 风险提示 | 时效风险、价格波动、节点拥堵、单证缺失、承运商履约风险。 |
| 推荐理由 | 用自然语言解释方案为何被推荐，列明关键数据依据。 |
| 置信度 | 根据数据完整性、历史样本、实时数据覆盖和模型稳定性给出等级。 |

## 9.5 智能引擎总体流程

1. 需求校验：检查起终点、吨位、煤种、时间、装卸、运输方式等必要字段。
2. 数据补全：调用节点、线路、承运商、历史运价和实时在途数据补齐计算依据。
3. 候选线路生成：基于地图服务、节点网络和运输方式组合生成候选路线。
4. 费用测算：按公路、铁路、短倒、装卸、仓储、等待和风险等成本项进行拆解。
5. 承运商匹配：结合线路覆盖、可用运力、资质、信用、履约和报价生成候选承运商。
6. 多目标评分：从成本、时效、风险、信用、政策适配度和数据完整性维度排序。
7. 方案解释：生成面向业务人员可读的推荐理由和风险说明。
8. 人工确认：关键订单、低置信度方案和高风险承运商需平台人员确认。
9. 结果回流：将成交、履约、异常、评价和成本结果回流模型，持续优化。

## 9.6 模型体系设计

| 模型或能力 | 技术方式 | 输出结果 |
|------------|----------|----------|
| 费用测算模型 | 规则引擎、历史运价拟合、实时报价修正。 | 分项成本、总成本、成本区间和差异原因。 |
| 线路生成模型 | GIS 路网、节点网络、多目标约束优化。 | 候选路线、运输方式组合、节点序列。 |
| 承运商推荐模型 | 信用评分、学习排序、规则过滤、可用运力匹配。 | 承运商推荐排序、推荐原因、风险等级。 |
| 运价预测模型 | 时间序列、回归模型、区域供需特征、季节因素。 | 重点线路运价趋势和价格区间。 |
| 异常预警模型 | 规则阈值、轨迹偏离检测、时效预测、单证完整性检查。 | 异常预警、风险类型、处理建议。 |
| 方案解释能力 | 知识库检索、模板化说明、自然语言生成。 | 推荐理由、方案说明、客户可读报告。 |

## 9.7 费用测算模型设计
费用测算模型应坚持“可拆解、可解释、可复核”的原则。系统既要给出综合物流成本，也要展示分项成本、计费依据、参考区间和不确定因素。

| 测算维度 | 计算依据 | 输出要求 |
|----------|----------|----------|
| 距离 | 地图服务距离、历史线路距离、铁路里程和短倒距离。 | 显示各运输段距离和合计距离。 |
| 吨位与煤种 | 运输需求吨位、煤种、装卸限制和车辆适配性。 | 显示吨位单价和煤种影响因素。 |
| 线路与节点 | 节点能力、装卸效率、拥堵、货场规则和专用线情况。 | 显示节点费用和时间影响。 |
| 运输方式 | 公路直达、铁路运输、公铁联运、多式联运、总包运输。 | 形成不同方式成本对比。 |
| 市场行情 | 历史成交价、承运商报价、区域供需、季节性和油价等变量。 | 形成价格区间和偏离提示。 |
| 风险修正 | 异常率、时效违约、天气管制、节点拥堵和承运商履约风险。 | 输出风险成本或风险说明。 |
## 9.8 线路生成与多式联运优化

线路生成应以节点网络为基础，将煤矿、货场、铁路站点、园区、仓储和终端用户连接成可计算网络。系统按运输方式和业务约束生成候选线路，再进行成本、时效、风险和政策适配度综合评分。

- 公路直达方案：适合短中距离、时效要求高、节点转换成本高的场景。
- 铁路方案：适合大吨位、中长距离、稳定计划和具备铁路发到站条件的场景。
- 公铁联运方案：适合需要短倒接驳、铁路干线和末端配送组合的场景。
- 总包运输方案：适合多批次、多线路、跨区域、需要统一组织和统一责任的场景。

## 9.9 承运商智能推荐

承运商推荐不应仅按最低报价排序，而应综合资质合规、线路覆盖、可用运力、履约表现、服务质量、安全记录、价格竞争力和数据配合度。

| 评分维度 | 建议权重 | 说明 |
|----------|----------|------|
| 资质合规 | 15% | 经营许可、证照有效期、业务范围、黑名单状态。 |
| 履约能力 | 20% | 准时率、完成率、取消率、异常率、历史合作表现。 |
| 线路与运力匹配 | 20% | 服务区域、车辆类型、可用运力、铁路协同能力。 |
| 报价竞争力 | 15% | 报价与历史均价、区域行情和预算区间的偏离度。 |
| 服务质量 | 10% | 投诉、纠纷、客户评价、单证上传及时率。 |
| 安全与合规记录 | 10% | 事故、违规、风险事件、安全培训和监管记录。 |
| 数据配合度 | 10% | 轨迹完整率、节点回传及时率、单证完整率。 |

## 9.10 多目标评分与推荐规则

综合推荐方案建议采用多目标评分方式，不以单一最低成本作为唯一依据。建议默认评分口径为：成本 30%、时效 25%、承运商履约 20%、合规安全 15%、数据完整性 10%。权重应支持按客户场景配置，例如保供运输提高时效权重，大宗稳定运输提高成本权重，监管敏感线路提高合规安全权重。

## 9.11 可解释与人工复核机制

AI 推荐结果必须可解释、可追溯、可人工复核。系统应在推荐结果中展示关键数据依据、推荐原因、风险提示和置信度。对于低置信度、高金额、高风险、跨方式复杂联运或监管关注订单，应自动进入人工复核流程。

- 显示推荐依据：成本、时效、承运商信用、历史履约、节点能力和政策适配度。
- 显示不确定因素：缺失数据、外部接口未回传、样本不足、价格波动和线路限制。
- 支持人工调整：业务人员可调整线路、承运商、权重和约束条件，并保留调整原因。
- 形成闭环学习：成交结果、实际成本、实际时效和异常事件回流模型。

## 9.12 AI 治理与安全要求

| 治理事项 | 要求 |
|----------|------|
| 数据质量治理 | 建立字段完整性、准确性、及时性、唯一性和一致性检查。 |
| 模型版本管理 | 记录模型版本、训练数据范围、上线时间、效果指标和回滚方案。 |
| 效果评估 | 按月评估费用测算偏差、推荐采纳率、异常预警准确率和承运商推荐命中率。 |
| 权限控制 | 不同角色仅能查看授权范围内的数据和推荐结果。 |
| 操作留痕 | 记录模型输出、人工调整、审批意见和最终成交结果。 |
| 安全边界 | 模型不直接替代合同签订、金融审批和监管结论，关键决策需人工确认。 |

## 9.13 AI 建设分期

| 阶段 | 建设重点 | 效果目标 |
|------|----------|----------|
| 一期 | 规则引擎、基础费用测算、候选线路生成、承运商规则推荐、方案说明。 | 实现可用、可解释、可人工复核的方案生成。 |
| 二期 | 运价预测、学习排序、信用评分、异常预警、模型效果评估。 | 提升推荐准确性和运营效率。 |
| 三期 | 跨平台数据融合、预测性调度、自动报告、行业指数和监管辅助分析。 | 形成数据产品和生态服务能力。 |
